# GPT 手动分析提示词

请读取这个仓库的 `items/**/meta.json` 和 `items/**/transcript.md`，只分析还没有有效 `analysis.md`/`analysis.json` 的条目。

目标：把每条内容整理成可回写飞书多维表格的结构化分析。不要调用付费 API，不要自动发布，不要改视频或音频文件。

请在每个条目的 `analysis.md` 里写入一个 JSON 代码块，字段必须使用下面这些英文 key：

```json
{
  "target_audience": "这条内容最适合影响/服务的人群",
  "topic": "选题或主题，一句话",
  "hook": "开头钩子或最值得复用的开场方式",
  "structure": "内容结构，用 3-6 个步骤概括",
  "pain_points": "击中的痛点，可以用分号分隔",
  "reusable_direction": "对我可复用的方向，比如小红书选题、skill素材、工作流优化、文案结构、账号定位",
  "value_rating": "高/中/低",
  "highlight": "爆点或可借鉴点",
  "tags": "标签，用分号分隔"
}
```

判断口径：

- 如果内容里出现可复用工具、自动化流程、skill、平台玩法，优先识别为 `skill素材` 或 `工作流优化`。
- 如果内容适合我做小红书账号选题，`reusable_direction` 里必须包含 `小红书选题`。
- `value_rating` 只允许写 `高`、`中`、`低`。
- 不要写长篇感想，字段要适合回写到飞书看板。

完成后把修改提交到仓库即可。本地工作台会读取这些分析结果并同步回飞书。
