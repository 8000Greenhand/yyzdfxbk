# 选题价值门禁

本文件用于防止工作台把“能做”误判成“值得发”。

## 1. 核心原则

`ready` 不等于“用户有材料、两小时能做完”。

一个选题只有同时满足以下条件，才能标记为 `ready`：

1. **有信息增量**：用户看完能得到一个原来不知道的机制、判断、方法或顺序；
2. **不是常识复述**：结论不能只是“AI不能替代人”“工具可以提效”“内容要有价值”“数据很重要”；
3. **能解释为什么**：不仅说结论，还要讲清导致结果的具体机制；
4. **能改变行动**：用户看完以后知道下一步该怎么做、先做什么、不要做什么；
5. **有真实证据**：至少有用户自己的案例、截图、数据、操作、失败或前后对比之一；
6. **与原素材有明显增量**：不是换标题复述原视频，也不是把原作者结论换成第一人称；
7. **适合当前账号**：内容能体现用户真实学习、实测和判断，而不是装成自己并不具备的专家。

## 2. 一票否决

出现以下任一情况，不得标记为 `ready`：

- 观众只看标题就已经知道全部结论；
- 结论正确但没有新解释、新证据或新方法；
- 只能靠“AI神器、太牛了、效率翻倍”等情绪包装；
- 用户本人无法提供真实案例；
- 内容只是解释工具分工，没有解决具体问题；
- 选题依赖用户尚未理解或尚未验证的知识；
- 与用户近期已发内容高度重复，但没有新的独立价值。

## 3. 选题升级方式

一个常识结论不能直接做选题，但可以作为背景，继续追问：

```text
常识：AI不能替人做决定
继续追问：为什么功能完整的AI工作台仍然会空、泛、没人打开？
具体机制：先做了页面，后补判断标准；隐性经验没有转成规则和样例
行动方法：先人工跑通3—5个真实案例，再提炼输入、判断、输出和异常，最后让Codex产品化
真实证据：工作台完成后多个模块为空；补齐analysis.md后才出现内容
```

只有追问到机制、行动和证据层，才形成真正值得发布的选题。

## 4. 进入选题池前的四问

1. 观众为什么不能只靠常识得到这个答案？
2. 这条内容新增的具体机制或判断是什么？
3. 看完后，观众会改变哪个动作？
4. 用户能展示什么真实证据？

四问中任何一项答不出来，状态改为：

- `needs_learning`；
- `needs_research`；
- `needs_test`；
- `not_fit`；
- 或不进入选题池。

## 5. 工作台显示规则

“今日行动”和“可制作”区域不得只按 `readiness == ready` 排序，还要确认：

- `why_worth_testing` 中明确写出信息增量；
- 有至少一个可展示证据；
- 不是一票否决中的常识题；
- 人工可以把该选题降级、隐藏或退回学习。

## 6. 本次纠错

已否决旧选题：

> 为什么Codex能帮我做工作台，却不能替我决定账号该发什么？

原因：结论停留在“AI不能替人决定”的常识层，观众只看标题就已经知道答案。

升级后的有效问题是：

> 为什么很多AI工作台功能都做完了，最后仍然空、泛、没人愿意打开？

它提供的新信息是：真正的问题通常不是代码能力，而是把页面做在了判断标准之前；正确顺序是先人工跑通真实案例，再把成熟规则交给Codex产品化。
