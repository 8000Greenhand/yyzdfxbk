# 分析结果

## 1. 原视频内容分析

### 内容总结

盘点自媒体从选题调研、文案脚本、内容包装到发布复盘的 AI Skill 工具链。

### 标题分析

标题用“自媒体人的 Skill 盘点”“从调研到复盘”“一条视频全讲透”制造完整清单感。它不是单点技巧，而是承诺给一套流程，因此收藏动机强。标题里的“附文档领取”也会增强评论和私信动作。

### 开头分析

开头用“如果现在让你马上做一条视频，你脑子里的第一步是什么？”让观众自检。这个问题能迅速暴露很多人的真实状态：没有流程、靠灵感、临时拼凑。

### 内容结构分析

1. 先用提问切入内容生产流程混乱的问题。
2. 讲选题调研：好选题不是灵感，是数据、爆款内容和评论区反馈长出来的。
3. 讲脚本生产：标题、前三秒、卖点、转场、互动引导都可以拆成模块。
4. 讲包装生产：字幕、配音、画面、剪辑、素材调用可以工具化。
5. 讲发布复盘：看数据、分析评论、更新策略。
6. 继续扩展风格蒸馏、自升级、安全审查、版权检查、内容日历等 Skill。

### 爆点分析

- **点击点**：自媒体人、Skill盘点、从调研到复盘。
- **收藏点**：完整清单型内容，适合收藏后照着搭流程。
- **评论点**：文档怎么领取、有哪些工具、Skill怎么写。
- **真正有价值的核心**：不是列工具，而是把内容生产拆成固定环节，每个环节都可以沉淀为一个可复用 Skill。

## 2. 工具 / 技术 / 资源

原视频更偏 Skill 体系盘点，未明确给出完整工具清单。真正值得拿走的是这些 Skill 方向：

- **选题调研 Skill**：输入主题/账号方向，输出用户问题、同行选题、可做角度。
- **爆款拆解 Skill**：输入爆款内容，输出标题钩子、开头、结构、收藏点、互动点。
- **评论区选题 Skill**：输入评论区内容，输出高频问题和下一条选题。
- **反AI味文案检查 Skill**：检查 AI 稿里的模板感、空话、过度升华。
- **标题封面 Skill**：根据选题生成标题、封面大字和点击理由。
- **发布复盘 Skill**：输入发布数据，输出问题归因和下一条测试方向。
- **内容日历 Skill**：把选题池排成一周/一月发布计划。
- **安全审查 Skill**：检查版权、敏感词、平台风险表达。

## 3. 给我的内容改造方案

### 方案 A：别再收藏AI工具了，打工人真正该先做这5个工作流

- **可改造方向**：不要照搬“自媒体人Skill盘点”，而是改成普通职场人能用的 AI 工作流清单。
- **标题备选**：
  1. 别再收藏AI工具了，打工人真正该先做这5个工作流
  2. 普通职场人用AI，先别学复杂的，先自动化这5件事
  3. 我发现AI提效最有用的不是写文案，而是做工作流
- **封面文案**：
  - 打工人AI提效
  - 先做这5个工作流
- **开头口播**：
  很多人学AI的第一步是收藏工具，但我后来发现，真正提效的不是工具，而是把你每天重复做的事变成固定流程。
- **内容结构**：
  1. 先讲“收藏工具没用”的痛点。
  2. 提出“工作流比工具重要”。
  3. 讲5个工作流：反AI味检查、标题封面生成、爆款拆解入库、工作资料沉淀、重复表格处理。
  4. 每个说明输入、输出、适用场景。
  5. 结尾引导评论：你最想让AI帮你处理哪件重复工作？

### 方案 B：我把小红书日更拆成了几个 AI Skill

- **可改造方向**：展示你自己如何把日更流程拆成可复用模块。
- **标题备选**：
  1. 我把小红书日更拆成了这几个AI Skill
  2. 做小红书别只问AI写稿，先把流程拆开
  3. 我终于不用每天从0开始想内容了
- **封面文案**：
  - 小红书日更
  - 拆成AI Skill
- **开头口播**：
  我以前以为用AI做小红书，就是让它帮我写文案。后来发现这样很难稳定，真正有用的是把每一步拆成 Skill。
- **内容结构**：选题判断 → 标题封面 → 正文结构 → 去AI味检查 → 发布复盘 → 评论区选题。

### 可补充资料 / 延伸学习

可以继续补充“小红书账号冷启动复盘字段”“评论区选题方法”“反AI味文案清单”“标题封面生成模板”。这类资料比继续搜新工具更有用。
