# 短视频素材工作台 V3 · 最终产品规范

## 0. 产品目标

这不是一个“大而全的内容决策系统”，也不是五个复杂资料库。

工作台只需要稳定完成五件事：

1. 把飞书和千瓜收录的链接处理清楚；
2. 把每天值得关注的 AI 信息放进编辑部；
3. 把素材和编辑部里真正有用的知识分门别类沉淀下来；
4. 选一个内容方向，再参考真实爆款标题和封面完成改写；
5. 记录发布结果，知道哪些内容和表达以后值得继续。

页面必须简洁、清楚、可直接使用。不要为了“专业感”堆过多字段、卡片和流程名。

---

## 1. 页面顶部：收录数据条

页面标题仍为“短视频素材工作台”。

标题下面放一条高度较小的横向数据条，只显示：

- 总收录链接：X 条；
- 飞书收录：X 条｜今日新增 X 条；
- 千瓜收录：X 条｜今日新增 X 条。

要求：

- 横向排列；
- 卡片较小，不占首屏主体；
- 点击“飞书收录”或“千瓜收录”可以筛选对应来源；
- 今日新增按本地日期与 `collected_at/exported_at` 计算；
- 不能识别来源的旧数据计入总数，并显示“来源待补”，不得擅自归到飞书或千瓜。

推荐来源字段：

```json
{
  "ingestion_source": "feishu|qiangua|manual|unknown",
  "collected_at": "ISO datetime"
}
```

Codex需要先检查现有飞书、千瓜导入链路，尽量在导入时写入来源；旧数据无法判断时允许用户手动修改并保存在本地状态中。

## 2. 五个一级板块

固定顺序：

1. **链接处理**
2. **AI编辑部**
3. **知识库**
4. **选题库**
5. **复盘库**

全局搜索保留，可搜索标题、作者、分类、工具、Skill、平台、知识名称、选题和来源。

# 3. 板块一：链接处理

## 3.1 作用

这里负责处理所有从飞书、千瓜或手动加入的链接。

每条素材完成三件事：

1. 判断它属于哪一类；
2. 展示逐字稿和原内容分析；
3. 提取可沉淀技能/知识和可复刻爆款表达。

它不是知识库。原素材永远保留在这里；提取出的知识再进入板块三。

## 3.2 默认分类

左侧分类固定先提供：

- **Skill / 工具**：分享 Skill、插件、网站、软件、平台功能、GitHub 项目或具体工具；
- **AI领域知识**：模型更新、产品功能、额度、价格、规则、使用区别和 AI 行业硬核知识；
- **运营知识**：内容运营、小红书、达人合作、数据分析、选题、标题、投放等；
- **职场效率**：PPT、表格、文案、会议、资料整理、自动化和工作技巧；
- **个人成长**：学习方法、职业成长、认知和经验；
- **爆款参考**：内容本身没有值得沉淀的硬核知识，但标题、封面、开头或结构数据很好；
- **待分类**：AI无法确定，留给用户手动处理。

允许用户新增分类、修改分类和批量分类。

分类原则：

- 一条素材只设一个主分类；
- 可另外打多个标签；
- 如果既有硬核知识又是爆款，主分类按“内容主要在讲什么”确定，同时标记“可作爆款参考”；
- 不要因为出现 AI、工作流、Skill 等词就强行分类，要根据逐字稿的主要内容判断。

## 3.3 分类列表页

点击左侧分类后，中间列表显示该类素材。

每条素材只显示：

- 封面；
- 标题；
- 一句话总结：这条内容主要讲什么；
- 点赞、收藏、评论、分享或互动量中已有的数据；
- 收录来源：飞书 / 千瓜 / 手动 / 待补；
- 内容平台：小红书 / 抖音 / 其他；
- 发布时间；
- 是否含可沉淀知识；
- 是否含爆款复刻资产。

不要在列表卡片堆长分析、验证等级和复杂路由。

## 3.4 素材详情页

打开一条素材后，顶部只显示：

- 原链接；
- 收录来源；
- 内容平台；
- 发布时间和现有互动数据；
- 分类修改入口。

不要再次重复大标题和一句话总结。

详情分四个区块：

### A. 逐字稿

- 展示完整 `transcript.md`；
- 默认折叠一部分，支持展开、复制；
- 没有逐字稿时显示“暂未转写”，不能报错。

### B. 原内容分析

只展示用户真正需要的分析：

- 标题与封面为什么有效；
- 前 3—5 秒怎么开；
- 内容按什么顺序展开；
- 为什么会获得点击、收藏、评论或分享；
- 这套表达成立的条件；
- 哪些地方不能照搬。

### C. 沉淀技能 / 知识

这里只展示该素材提取出的事实性、有利用价值的内容，例如：

- Skill；
- 工具；
- 网站；
- 平台；
- 插件；
- GitHub 项目；
- 产品功能；
- 具体规则、设置、额度和经过核验的知识。

每条只在第一屏显示：

- 名称；
- 一句话：它是干什么的；
- 当前状态：二手提及 / 已找到来源 / 已实测；
- 是否已进入知识库。

点击后再展开：怎么用、入口、限制、来源和下一步。

抽象的“抓取→分析→分类”或“这个工作流很好”不得作为独立知识沉淀。

### D. 爆款复刻

每条素材最多输出：

- 1—3 个可复刻标题；
- 1—3 个可复刻开头；
- 原内容结构；
- 干货点；
- 价值锚点；
- 信任证据；
- 适用主题；
- 照搬风险。

这一块服务于板块四的选题和改写，不在这里生成大量脚本。

# 4. 板块二：AI编辑部

AI编辑部继续承担每日重要信息处理，不需要重做成复杂新闻网站。

保留四个子区：

1. **今日判断**：今天是否有必须关注的更新，今天最值得做什么；
2. **更新拆解**：具体更新、位置、新旧区别、额度、价格、地区、账号门槛、普通用户影响；
3. **搜索需求**：用户正在搜什么、哪些问题缺少好答案；
4. **待核验**：缺官方来源、缺截图、缺实测的问题。

编辑部输出可以同时流向：

- 知识库：沉淀可长期使用的知识；
- 选题库：产生可做的选题；
- 不值得长期保留的信息只留在当日简报。

推荐读取：

`editorial/daily/YYYY-MM-DD.json`

没有文件时显示空状态，不允许调用 Codex 定时抓取新闻。

# 5. 板块三：知识库

## 5.1 核心原则

知识库不是长篇分析页，也不是 AI 百科。

它应该像一份干净、清楚、可以随时查用的个人知识清单。

知识来源只有两类：

- 板块一“链接处理”中提取的技能和知识；
- 板块二“AI编辑部”中核验后值得长期保留的知识。

## 5.2 默认分类

左侧默认分类：

- **Skill沉淀**；
- **AI领域知识**；
- **运营知识**；
- **职场效率**；
- **个人成长**；
- **待分类**。

允许新增自定义分类。

其中“Skill沉淀”可以用子标签筛选：Skill、工具、网站、平台、插件、GitHub项目、产品功能。

## 5.3 列表呈现

列表要紧凑、简练，不使用大卡片。

每条只显示：

- 名称；
- 一句话用途或结论；
- 类型小标签；
- 来源数量；
- 最近更新时间；
- 状态：二手提及 / 来源已确认 / 已实测。

示例：

```text
FunASR｜本地把视频或音频批量转成中文逐字稿｜工具｜已找到来源
GPT模型选择｜简单任务用极速，普通任务用中，复杂决策用高｜AI知识｜待持续更新
```

## 5.4 知识详情

点击后清楚显示：

- 它是什么；
- 它解决什么问题；
- 用户可以怎么利用；
- 使用方法或核心结论；
- 入口、账号、代码、API、费用和限制；
- 原始来源；
- 最后核验时间；
- 关联素材；
- 关联选题。

同一个知识只保留一条，通过来源列表合并，不重复堆积。

无法判断分类的知识进入“待分类”，允许用户手动移动。

# 6. 板块四：选题库

## 6.1 真实用途

这里不是自动给用户塞一堆完整脚本，而是帮助用户完成：

> 先选“我今天想写什么”，再选“我想参考哪个真实爆款”，最后得到一组可用的标题、封面和开头改写。

## 6.2 页面布局

采用三栏结构。

### 左栏：选题方向

按知识分类展示可写方向：

- AI领域；
- Skill / 工具；
- 运营；
- 职场效率；
- 个人成长；
- 自定义分类。

选择分类后显示该类可用选题。每条只显示：

- 选题名称；
- 一句话：这条内容要解决什么问题；
- 来源：编辑部 / 知识库 / 人工新增；
- 状态：待补资料 / 可制作 / 已发布。

### 中栏：爆款参考

选中一个选题后，列出与该主题匹配的真实爆款素材：

- 封面；
- 原标题；
- 点赞、收藏、评论、分享或互动；
- 平台和发布时间；
- 该案例为什么适合作为参考的一句话；
- 参考按钮。

数据来自板块一中标记为“可作爆款参考”的素材。

允许用户手动选择一个参考案例，不由系统强行决定。

### 右栏：参考与改写

选择参考案例后，右侧显示：

- 原封面；
- 原标题；
- 原开头；
- 可复用的标题/封面组合逻辑；
- 成立条件和照搬风险；
- 改写方案 1—5。

每个改写方案包含：

- 标题；
- 封面主标题；
- 封面副标题；
- 前 3 秒开头；
- 一句话说明为什么这样改。

支持：

- 收藏某个改写；
- 保存到“待制作”；
- 标记“不要”；
- 复制内容。

## 6.3 V1生成边界

本地静态工作台在不接付费 API 的情况下，不能根据任意“新选题 + 任意爆款案例”实时生成高质量改写。

因此 V1 必须采用：

1. 优先显示 `analysis.md` 或编辑部文件中已经生成的改写；
2. 没有现成改写时，自动组装一段完整“改写指令”，提供“复制到 ChatGPT”按钮；
3. 用户把 ChatGPT 返回的 5 组结果保存为本地数据或后续同步文件；
4. 不允许用简单字符串替换伪装成 AI 改写。

未来只有用户明确同意接 API 或本地模型时，才增加工作台内实时生成。

# 7. 板块五：复盘库

继续保留，但保持简单。

列表显示：

- 发布内容；
- 对应选题；
- 参考爆款；
- 实际标题、封面和开头；
- 发布时间；
- 曝光、播放/阅读、点赞、收藏、评论、分享、主页访问、关注；
- 搜索词或搜索流量（如有）；
- 一句话复盘。

详情回答：

- 哪个环节有效；
- 哪个环节失败；
- 下次保留什么；
- 下次修改什么；
- 是否继续这个选题方向或表达方式。

V1可继续使用 localStorage，并支持导入、导出 JSON。

# 9. 页面体验

- 保留现有白底、蓝色、简洁风格；
- 数据条小而清楚；
- 板块一和板块三都使用左侧分类；
- 板块四采用明确三栏布局；
- 长逐字稿和长分析折叠；
- 不在卡片首屏展示长脚本；
- 1366px和1920px宽度均可正常使用；
- 本地直接打开 `workbench.html` 可运行；
- 不引入不必要的大型框架和在线依赖。

# 10. 最终验收标准

1. 顶部能看到总收录、飞书收录/今日新增、千瓜收录/今日新增；
2. 五个板块名称和顺序正确；
3. 链接处理可按默认分类和自定义分类查看；
4. 素材列表只显示封面、标题、总结、互动、来源和平台等关键信息；
5. 素材详情按“逐字稿、原内容分析、沉淀技能/知识、爆款复刻”展示；
6. 用户可以手动修改分类和来源；
7. AI编辑部继续正常展示且不由Codex定时抓新闻；
8. 知识库使用简洁列表，能合并同一知识的多个来源；
9. 知识可按Skill沉淀、AI领域知识、运营、职场效率、个人成长和待分类查看；
10. 选题库实现左侧选题、中间爆款参考、右侧参考与改写的三栏布局；
11. 没有预生成改写时提供“复制到ChatGPT”的完整指令，不做伪AI生成；
12. 复盘库可录入、编辑、导入和导出；
13. V3、V2和legacy分析都能兼容，单条错误不影响全页；
14. 原GitHub同步、中文路径、逐字稿和旧素材不被破坏；
15. 不接付费API、不自动发布、不持续消耗Codex处理日常内容。
